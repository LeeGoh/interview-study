## [ Java ] OOP 특징

🔩 **OOP**

---

객체 지향 프로그래밍 Object Oriented Programing

객체의 관점에서 프로그래밍을 하는 것

프로그래밍에서 필요한 데이터를 추상화시켜 상태와 행위를 가진 객체를 만들고 그 객체들 간의 유기적인 상호작용을 통해 로직을 구성하는 프로그래밍 방법

<br><br>

+ **OOP 장점**

---

1. **코드 재사용 용이**
    
    이미 만들어져 있는(또는 다른 사람이 만든) 클래스를 가져와서 사용할 수 있고, 상속을 통해 확장시킬 수 있음
    
2. **쉬운 유지보수**
    
    객체 지향 프로그래밍에서는 수정해야 할 부분이 클래스 내부에 멤버 변수혹은 메서드로 존재하기 때문에 해당 부분만 수정할 수 있음
    
3. **대형 프로젝트에 적합**
    
    클래스 단위로 모듈화시켜서 개발할 수 있기 때문에 대형 프로젝트처럼 여러 명, 여러 회사에서 프로젝트를 개발할 때 업무 분담하기에 용이함
    

<br><br>

+ **OOP 단점**

---

1. 상대적으로 느린 처리 속도
2. 객체가 많아지면 용량이 커질 수 있음
3. 설계 시 많은 시간과 노력이 필요

<br><br>

🔩 **OOP 특징 - 캡슐화**

---

1. 캡슐화
2. 추상화
3. 상속
4. 다형성

<br><br>

🔩 **캡슐화**

---

객체 지향 프로그래밍에서 기능과 특성의 모음을 클래스라는 캡슐에 분류해서 넣는 것

1. 코드 수정 없이 재활용 가능
2. 접근 제어자를 통한 정보 은닉
3. 높은 응집도, 낮은 결합도

<br><br>

🔩 **추상화**

---

공통의 속성이나 기능을 묶어 하나의 개념(집합)으로 다루는 것/이름을 붙이는 것

목적과 관련없는 부분은 제거하고 필요한 부분만을 표현하기 위해 사용

<br><br>

🔩 **상속**

---

기존 상위클래스의 속성과 기능을 바탕으로 새로운 클래스와 행위를 재정의 하는 것

상위클래스의 기능 일부분 변경 필요 시 용이 → 불필요한 수정없이 해당 기능만 재정의 가능

<br><br>

🔩 **다형성**

---

형태는 같지만 다른 기능을 하는 것

함수명, 변수명이 같아도 다른 기능을 할 수 있음

- **오버라이딩**: 상위클래스의 메서드와 같은 이름 또는 인스턴스를 재정의 하는 것
- **오버로딩**: 같은 이름의 함수를 여러개 정의 후 인스턴스의 타입과 개수를 다르게 하여 인스턴스별로 다르게 호출할 수 있도록 하는 것

<br><br>

+ **클래스/인스턴스**

---

**클래스**

객체를 만들기 위한 메타정보

문제 해결을 위한 데이터를 만들기 위해 추상화를 거쳐 속성과 행위를 변수와 메서드로 정의한 것

<br>

**인스턴스**

실제 프로그램에서 사용되는 데이터

클래스에서 정의한 것을 바탕으로 실제 메모리에 할당된 것

<br><br>

🔩 **예비 답변**

---

Q. OOP 객체 지향 프로그래밍에 대해 설명해주세요.

OOP 객체 지향 프로그래밍이란 프로그래밍에서 필요한 데이터를 추상화시켜서 상태와 행위를 가지고 있는 객체를 만들고, 그 객체들 간의 유기적인 상호작용을 통해서 로직을 구성하는 즉, 객체의 관점에서 프로그래밍을 하는 것입니다.

객체 지향의 장점으로는 코드의 재사용이 용이하고, 유지보수가 쉽다는 점이 있고, 단점으로는 상대적으로 처리속도가 느리다는 것과 객체가 늘어남에 따라 용량도 증가한다는 점이 있습니다.

<br>

Q. OOP 객체 지향 프로그래밍의 특징에 대해 설명해주세요.

객체 지향의 특징으로는 캡슐화, 추상화, 상속, 다형성이 있습니다.

**캡슐화**는 하나의 객체에 대해 목적을 위해 필요한 변수나 메서드를 하나의 캡슐로 묶는 것입니다. 코드를 재사용할 수 있고 접근제어자를 통한 정보 은닉이 가능합니다.

**추상화**는 목적과 관련이 없는 부분은 제거하고 공통된 특징을 파악하여 필요한 부분만을 정의하는 설계 기법입니다. 

**상속**은 기존 상위 클래스를 바탕으로 새로운 클래스와 행위를 재정의하는 것입니다. 코드의 중복을 없앨 수 있고, 기존의 코드를 수정하지 않고도 새로운 기능을 추가할 수 있습니다.

**다형성**은 변수 명, 메서드 명 등 형태는 같지만 다른 기능을 하는 것입니다. 코드를 재사용할 수 있고, 코드의 길이가 감소되어 유지보수가 용이합니다. ( + 대표적인 예시로는 오버라이딩과 오버로딩이 있습니다.)
